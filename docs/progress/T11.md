#### Quyết định kỹ thuật
- `AuthUser` contains both `user_id: Uuid` and raw `claims: JwtClaims` for downstream access to email, username, roles.
- `AuthState` (holding `JwtValidator`) is injected via `axum::Extension` layer, not axum `State`. This allows the extractor to work with any route without requiring `State<AuthState>` in the handler signature.
- `AuthError` implements `IntoResponse` with proper HTTP status codes: 401 for auth failures, 503 for JWKS fetch failures.
- `UserNotFound` variant is kept for future use in T12+ when we need to verify user exists in DB.

#### Blocker / Warning cho batch tiếp theo
⚠️ `claims` field on `AuthUser` is currently unused — will be used when routes need email/username/roles.
⚠️ `with_cache_ttl` method on `JwtValidator` is unused — available for tests that need custom TTL.
