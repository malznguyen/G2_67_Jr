#### Quyết định kỹ thuật
- Applied `infra/postgres/init.sql` manually before migration because `gmrag_current_tenant()` function was not in the DB (the `vector` extension warning is expected — it's a soft dependency).
- `tenant_members` has RLS enabled with policy `tenant_members_isolation` filtering by `gmrag_current_tenant()`.
- `users` and `tenants` tables do NOT have RLS yet — they are global reference tables. RLS will be added when tenant-scoped operations are needed.
- `platform_admins` is a global table (no tenant_id column) for super-user management.
- Used `gen_random_uuid()` from `pgcrypto` extension for default UUIDs.
- Added `idx_users_email` index for fast email lookups and `idx_tenant_members_user` for reverse membership queries.

#### Blocker / Warning cho batch tiếp theo

⚠️ `platform_admins` table has no RLS — it's intentionally global.
