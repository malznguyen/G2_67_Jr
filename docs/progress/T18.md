#### Quyết định kỹ thuật
- **Handler requires PgPool via Extension:** The handler extracts `AuthUser` (JWT auth) and `Extension<PgPool>` (database). If PgPool is not in extensions, axum returns 500 automatically.
- **Does NOT require TenantContext:** This endpoint returns the user's own data and their tenant memberships. No tenant-scoped filtering is needed.
- **Uses sqlx::query_as! (compile-time verified):** The SQL queries are verified at compile time against the database schema. Requires `DATABASE_URL` env var during compilation.
- **Response shape:** `{ "user": { "id", "email", "name", "created_at" }, "tenants": [{ "id", "name", "role" }] }` — consistent with the JSON envelope pattern.
- **Error handling:** Uses `ApiError` (from T17). Returns `NotFound` if user doesn't exist in DB, `Internal` for DB errors.
- **Route mounted at `/users/me`:** Part of the main router. In production, this route should be behind auth middleware (AuthState must be in extensions).

#### Blocker / Warning cho batch tiếp theo
⚠️ `PgPool` and `AuthState` must be added to request extensions in main.rs for the `/users/me` route to work in production. Currently, main.rs does NOT add these extensions — only the router routes are mounted.
⚠️ The `sqlx::query_as!` macro requires `DATABASE_URL` env var at compile time pointing to a running PostgreSQL with the correct schema. Without it, `cargo build` fails.
