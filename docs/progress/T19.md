#### Quyết định kỹ thuật
- **Two-pool design (R1 fix):** `init_pool` (superuser `gmrag`, bypass RLS) dùng cho migrate + platform/cross-tenant ops; `init_app_pool` (after_connect `SET ROLE gmrag_app`, RLS enforced) dùng cho `rls_middleware`. Migrations chạy trên `admin_pool` vì `gmrag_app` thiếu `CREATE` trên schema.
- **Newtype wrappers `AdminPool`/`AppPool`:** Cần thiết vì axum extension map dùng `TypeId` — không thể giữ 2 `Extension<PgPool>` cùng lúc. Cả hai deref tới `PgPool`.
- **Cả AuthUser + TenantContext thành middleware (R2 fix):** Chain `auth_middleware → tenant_middleware → rls_middleware → handler`. Mỗi middleware populate extension cho middleware kế tiếp. Handler args đổi sang `Extension<AuthUser>`/`Extension<TenantContext>`/`Extension<SharedConnection>`.
- **tenant_middleware dùng AdminPool cho membership check:** Lookup `tenant_members` phải chạy TRƯỚC khi RLS context được set — dùng admin pool (bypass RLS) tránh chicken-and-egg. Sau khi T25 add RLS policy lên `tenant_members`, app pool sẽ hide row cần authorize.
- **get_me dùng AdminPool:** Cross-tenant (list ALL memberships) — preserved behavior. RLS trên `tenant_members` sẽ hide memberships nếu dùng app pool.
- **tenant_scoped group không mount route:** Middleware chain được wire (compile + dùng) nhưng chưa có handler tenant-scoped thật (HOTFIX BLOCKER-3 carry). `#![allow(dead_code)]` giữ lại cho safety.
- **healthz đổi sang Extension<AdminPool>:** Trước dùng `State<AppState>.pool`; nay AppState không còn pool (pools là Extension). healthz ping DB qua admin pool.
