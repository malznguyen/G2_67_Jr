#### Quyết định kỹ thuật

1. **Relative bind path corrected** — Initial draft used `./infra/postgres/init.sql`, but compose resolves paths relative to the compose file's own directory, producing `infra/infra/postgres/init.sql`. Fixed to `./postgres/init.sql` (green phase verified).
2. **`infra/backend.Dockerfile` & `infra/frontend.Dockerfile` referenced but not created** — out of scope for T1-T4 (gmrag2/ + infra/ + .env.example). `docker compose config` only validates YAML, not Dockerfile existence. The actual Dockerfile creation belongs to later tasks (T5+). Backend/worker `build` blocks will surface a build error at `up` time, not at `config` time, which matches the task's "test = config passes" acceptance.
3. **`gmrag-net` named bridge network + named volumes** — all 9 services share one network and persistent volumes are prefixed `gmrag-*` to avoid clashes with other local stacks.
4. **Tenant scaffolding in `init.sql`** — created `gmrag_current_tenant()` reading `app.tenant_id` GUC + a least-privilege `gmrag_app` role. Domain tables and RLS policies are intentionally deferred to migration tasks.
5. **Keycloak healthcheck** — uses raw TCP/HTTP probe against `/health/ready` (start-dev mode exposes it on `:9000`); avoids requiring `curl`/`wget` in the image.
6. **MinIO init is idempotent** — `mc ls` guards re-creation; safe to re-run.
