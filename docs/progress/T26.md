#### Quyết định kỹ thuật
- **UUIDs hex-valid only:** Tất cả UUID dùng hex chars (a-f, 0-9) — prefix `a1` (tenants), `b1` (users), `c1` (workspaces), `d1` (documents), `e1` (chunks), `f1`/`f2` (chat), `a2` (qdrant points), `b2` (audit), `c2` (ingest jobs). Tránh lỗi `invalid input syntax for type uuid` như lần chạy đầu (prefix `u`, `w` không phải hex).
- **Idempotent (ON CONFLICT DO NOTHING):** Seed có thể chạy nhiều lần — `ON CONFLICT (id)` cho UUID PK tables, `ON CONFLICT (tenant_id, user_id)` cho tenant_members, `ON CONFLICT (workspace_id, user_id)` cho workspace_members, `ON CONFLICT (tenant_id)` cho tenant_quotas.
- **BEGIN/COMMIT wrapper:** Toàn bộ seed trong 1 transaction — nếu 1 INSERT fail, toàn bộ rollback (atomic seed).
- **Seed chạy as superuser (gmrag):** Bypass RLS — seed insert data cho ALL tenants. Runtime, app pool (gmrag_app) chỉ thấy rows matching `app.tenant_id`.
- **.sqlx offline cache:** `cargo sqlx prepare --workspace` sinh 2 JSON files cho `query_as!` trong `routes/users.rs`. Verified: `SQLX_OFFLINE=true cargo build --workspace` PASS không cần DATABASE_URL. CI/developers khác có thể build mà không cần DB sống.
- **Không cần bless checksum (T25/T26):** Không có `.sqlx` cache cũ → tạo fresh. Migration T12/T15 checksum bless từ HOTFIX_BATCH2B vẫn valid (files không bị sửa trong batch này).
