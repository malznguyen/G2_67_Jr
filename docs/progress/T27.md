#### Quyết định kỹ thuật
- **`qdrant-client = "=1.12.1"` (pin exact):** Version `1.12.4` mà spec ngầm định (khớp server v1.12.4) **không tồn tại** trên crates.io (verify `cargo info qdrant-client@1.12.4` → ERROR "could not find in registry"). Dòng 1.12.x chỉ có `1.12.0` và `1.12.1` (Oct 2024). Pin `=1.12.1` (closest same-minor) — gRPC API ổn định trong cùng minor, đủ cho create/upsert/search/health.
- **gRPC port 6334, KHÔNG REST port 6333:** `qdrant-client` rust 1.12.1 dùng **gRPC exclusively** (verify docs.rs `QdrantConfig`: không có `use_http()` / REST toggle; tất cả doc examples dùng `http://localhost:6334`; `QdrantConfig::default()` = `http://localhost:6334`). Test ban đầu trỏ port 6333 (REST) → fail với `h2 protocol error / GoAway FRAME_SIZE_ERROR` (REST server không speak HTTP/2 gRPC framing). Fix: test config dùng `http://localhost:6334`. Cả 2 port đều mapped ra host (docker ps: `6333-6334->6333-6334/tcp`).
- **`Qdrant` không `Clone` → wrap `Arc`:** `qdrant_client::Qdrant` trong 1.12.1 không impl `Clone` (verify docs.rs). Wrap `Arc<qdrant>` để giữ `QdrantStore: Clone` (api/worker share 1 store cheap). Method calls dùng `&self` → Arc deref work.
- **`Error::Qdrant(Box<QdrantError>)` + manual `From`:** `QdrantError` ~176 bytes (wraps `tonic::Status` + variants) → inline vào `Error` enum trigger `clippy::result_large_err` (được `-D warnings` ép thành error). Box variant giữ `Error` nhỏ trên happy path. thiserror `#[from]` trên `Box<T>` chỉ generate `From<Box<T>>`, không phải `From<T>` — nên implement `From<QdrantError>` manually để caller vẫn `.map_err(Error::from)` trên `Result<_, QdrantError>`.
- **`new` async + health_check probe:** Khớp pattern `init_pool` (fail-fast `SELECT 1`). Test verify "kết nối local" được xác nhận thật, không chỉ construct client rỗng.
- **Test construct `QdrantConfig` trực tiếp (không đọc env):** Deterministic, isolated from operator `.env`. Không dùng `#[sqlx::test]` pattern vì không cần DB.
