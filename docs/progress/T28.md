#### Quyết định kỹ thuật
- **Collection naming `chunks_{tenant_id}`:** Theo spec — tenant-isolated collection. `tenant_id: Uuid` format trực tiếp vào tên (Qdrant collection name cho phép `_` + hex chars).
- **Vector config:** `VectorParamsBuilder::new(768, Distance::Cosine)` — single (unnamed) vector. HNSW dùng **Qdrant default** (spec yêu cầu "index HNSW" — Qdrant mặc định đã là HNSW cho dense vectors, không cần explicit `hnsw_config`). Verify test: `Config::Params(vp)` branch, `vp.size == 768`, `vp.distance() == Distance::Cosine`.
- **Payload index mapping (FieldType, verify 1.12.1 có đầy đủ variants):**
  - `workspace_id` → `FieldType::Uuid`
  - `document_id`  → `FieldType::Uuid`
  - `chunk_index`  → `FieldType::Integer`
  - `filename`     → `FieldType::Keyword`
  - `owner_id`     → `FieldType::Uuid`
  - `visibility`   → `FieldType::Keyword`
  - `FieldType::Uuid` tồn tại trong 1.12.1 (variant `Uuid = 7`, verify docs.rs) → map uuid fields trực tiếp, không cần fallback Keyword.
- **`.wait(true)` trên `CreateFieldIndexCollectionBuilder`:** Đảm bảo index được materialize trước khi return — tránh race condition khi upsert point ngay sau create (point land trước index tồn tại). Verify 1.12.1: method `wait(self, bool) -> Self` tồn tại.
- **Shared helper `create_collection_with_indexes`:** T28 + T29 cùng pattern (create 768/Cosine collection + attach payload indexes) → refactor thành private helper, T29 reuse với 3 indexes khác. Giảm duplication.
- **`delete_collection` semantics:** Wraps `client.delete_collection(DeleteCollectionBuilder::new(name))`. Qdrant trả error nếu collection không tồn tại — caller muốn "ensure gone" ignore `Err` (test cleanup dùng `let _ = store.delete_collection(&name).await;` cho stale cleanup).
- **Test verify qua `collection_info` (private field access):** Test trong cùng `mod tests` của `store.rs` → truy cập `store.client` (private `Arc<Qdrant>`) trực tiếp. Navigation path (verify 1.12.1): `collection_info(&name).await?.result` (`Option<CollectionInfo>`) → `.payload_schema: HashMap<String, PayloadSchemaInfo>` → `.data_type() -> PayloadSchemaType` (method, raw field là `i32`); `.config?.params?.vectors_config?.config?` → `Config::Params(VectorParams)` → `.size: u64` + `.distance() -> Distance`.
- **Cleanup-first pattern:** Test bắt đầu bằng `let _ = store.delete_collection(&name).await;` để dọn collection stale từ run trước (nếu test panic giữa chừng leak collection). Test sinh `Uuid::new_v4()` mỗi run → tên unique, không conflict.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* **`create_chunks_collection` không idempotent:** Gọi 2 lần với cùng `tenant_id` → lần 2 fail (collection đã tồn tại). Caller (worker provisioning) phải check `list_collection_names` trước hoặc bọc retry. Future task có thể thêm `ensure_chunks_collection` idempotent wrapper.
* **HNSW dùng default:** Không set explicit `hnsw_config` (m/ef_construct). Qdrant default (`m=16, ef_construct=100`) đủ cho dev. Production tuning thuộc task perf riêng.
* **Test leak collection nếu panic giữa chừng:** Cleanup-first ở đầu test dọn stale, nhưng collection hiện tại chỉ delete ở cuối test → nếu assert giữa chừng panic, collection đó leak. Acceptable cho dev Qdrant (re-run dọn stale). Không dùng `scopeguard` (chưa có dep).
* **T29 reuse `delete_collection` + `list_collection_names` + `create_collection_with_indexes`** — không cần thêm helper.
