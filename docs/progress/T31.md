#### Quyết định kỹ thuật
- **Signature `upsert_chunks(tenant_id, points: Vec<PointStruct>) -> Result<()>`:** Theo spec T31. Caller chịu trách nhiệm embed chunk text → vector + build payload (không để QdrantStore biết schema payload nội dung gì — chỉ transport). Trả `Result<()>` (discard `PointsOperationResponse`) vì caller không cần operation result cho 1 upsert.
- **`UpsertPointsBuilder::new(name, points).wait(true)`:** Dùng builder API mới (non-deprecated, `qdrant_client\points.rs:102`). `.wait(true)` đảm bảo upsert được acknowledge + index trước khi return — search ngay sau sẽ thấy point (critical cho T32 test recall).
- **Collection name = `chunks_{tenant_id}`:** Khớp T28/T30 convention. Không check `collection_exists` ở `upsert_chunks` — caller phải `setup_tenant_collections` trước (T30). Nếu collection chưa tồn tại, Qdrant trả error rõ ràng → caller biết thiếu provisioning.
- **Test provision qua `setup_tenant_collections` (T30 pub API):** Theo quyết định user. Test vừa verify T30 lifecycle vừa verify T31 upsert. Decoupled khỏi private `create_chunks_collection`.
- **Test verify qua `count(exact=true)` thay vì `get_points`:** `count` nhẹ hơn, đủ để chứng minh 3 point đã land. `exact=true` đảm bảo count chính xác (không estimate). `CountResponse.result: Option<CountResult>`, `CountResult.count: u64` (verify `qdrant.rs:5480-5504`).
- **Helper `unit_vec(pos)`:** Sinh vector 768-dim với `1.0` ở `pos`, `0.0` ở phần còn lại → cosine similarity deterministic (`1.0` cùng pos, `0.0` khác pos). Reuse cho T32 recall test.
- **Payload UUID lưu dạng string (`ws.to_string()`):** `Payload::try_from(serde_json::json!({"workspace_id": ws.to_string(), ...}))`. Qdrant lưu payload UUID dạng string keyword. Filter T32 sẽ dùng `Condition::matches("workspace_id", ws.to_string())` → `MatchValue::Keyword` (không có `MatchValue::Uuid` trong client 1.12.1 — verify `qdrant.rs:5725` enum chỉ có Keyword/Integer/Boolean/Text/Keywords/Integers/ExceptIntegers/ExceptKeywords).

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* **Filter UUID phải dùng keyword match (string):** `MatchValue` không có variant `Uuid` trong client 1.12.1. Payload `workspace_id` lưu string UUID; filter dùng `Condition::matches("workspace_id", ws.to_string())`. Payload index `workspace_id` đang là `FieldType::Uuid` (T28) — Qdrant Uuid-index **có** chấp nhận keyword match (string equality trên UUID string), nhưng nếu T32 test filter trả empty thì cần xem xét đổi index sang `FieldType::Keyword`. Watch ở T32 green step.
* **`upsert_chunks` không check collection tồn tại:** Caller phải `setup_tenant_collections` trước. Upsert vào collection không tồn tại → Qdrant error (`Collection `{name}` doesn't exist`). Acceptable — separation of concerns (provision = T30, ingest = T31).
* **Single-request upsert (không chunked):** `upsert_chunks` gửi tất cả point trong 1 gRPC request. Batch lớn (>1000 point) có thể timeout. Production nên dùng `upsert_points_chunked` (client có sẵn) hoặc split upstream — future task, ngoài scope T31.
* **`PointsOperationResponse` bị discard:** Caller không biết số point thực sự upserted (Qdrant trả `operation_id`/`status`). Acceptable cho happy path; nếu cần audit log, future task trả `Result<PointsOperationResponse>`.
