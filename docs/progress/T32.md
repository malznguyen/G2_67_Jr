#### Quyết định kỹ thuật
- **Signature theo spec:** `search_chunks(tenant_id, query_vector: Vec<f32>, filter: Option<Filter>, top_k: u64) -> Result<Vec<ScoredPoint>>`. Caller build filter (linh hoạt cho các filter combination khác nhau: workspace_id, document_id, visibility, owner_id). Trả `Vec<ScoredPoint>` đầy đủ payload (caller không cần `get_points` follow-up).
- **`SearchPointsBuilder::new(name, vec, top_k).with_payload(true)`:** Dùng builder API mới (`qdrant_client\search.rs:38`). `.with_payload(true)` include toàn bộ payload trong `ScoredPoint` → caller đọc `workspace_id`/`chunk_index` trực tiếp từ kết quả. Vector KHÔNG include (`.with_vectors(false)` default) → tiết kiệm bandwidth.
- **Filter attach có điều kiện:** `if let Some(f) = filter { req = req.filter(f); }`. `None` = unfiltered kNN (test sanity assertion dùng `None` để verify filter là thứ loại chunk 3, không phải point thiếu).
- **Filter UUID = keyword match (string):** `Filter::must([Condition::matches("workspace_id", ws_a.to_string())])`. `MatchValue` không có variant `Uuid` (verify `qdrant.rs:5725`) → match dạng `Keyword` (string equality). **CONFIRM WORKING:** payload index `workspace_id` là `FieldType::Uuid` (T28) nhưng Qdrant Uuid-index chấp nhận keyword match trên UUID string — test GREEN chứng minh filter trả đúng ws_a point, loại ws_b. Blocker T31 đã resolve.
- **Recall test design:** 3 chunk với vector trực giao (`unit_vec(0)`, `unit_vec(1)`, `unit_vec(0)`). Query A=`unit_vec(0)`: cosine(A,A)=1.0 (chunk 1 + chunk 3), cosine(A,B)=0.0 (chunk 2). Filter ws_a giữ chunk 1 (score 1.0) + chunk 2 (score 0.0), loại chunk 3 (ws_b). Top hit = chunk 1 (chunk_index 0) → recall đúng. Score sort: 1.0 ≥ 0.0 ✓.
- **Multi-tenant isolation 2 lớp:** `tenant_id` = collection boundary (hard isolation, mỗi tenant 1 collection riêng — T28/T30). `workspace_id` = payload filter (soft partition trong tenant). Caller API RAG query sẽ build filter từ TenantContext (không lấy tenant_id từ URL — nguyên tắc Bất Biến).
- **`with_payload(true)` vs `with_vectors`:** Payload include (cần cho business logic), vector exclude (không cần cho display, tiết kiệm). Nếu caller cần vector (vd. re-rank), future task thêm param `with_vectors: bool`.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* **T33 dùng cùng pattern nhưng cho `graph_{tenant_id}`:** `search_graph_nodes` nhận `workspace_id` trực tiếp (không `Option<Filter>`) và build filter nội bộ — khác signature T32 (caller build filter). Theo spec T33.
* **Score threshold chưa set:** `search_chunks` không set `score_threshold` → trả tất cả top_k point bất kể score (kể cả score 0.0 hoặc âm). Production RAG cần `score_threshold` để loại low-relevance hit — future task thêm param `score_threshold: Option<f32>`.
* **`top_k` không có offset:** Không support pagination. RAG thông thường top_k=5-20 đủ. Pagination (offset) thuộc future task nếu cần infinite scroll.
* **Read consistency default:** Không set `read_consistency` → Qdrant default (single-node dev = OK). Production multi-node cần `ReadConsistency::Majority` để tránh stale read sau upsert — future task.
* **HNSW default `ef` không tune:** Search dùng Qdrant default `hnsw_ef`. High-recall production cần `SearchParamsBuilder::default().hnsw_ef(128)` — future perf task.
