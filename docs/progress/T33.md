#### Quyết định kỹ thuật
- **`upsert_graph_nodes` mirror `upsert_chunks` (T31):** Cùng signature `(tenant_id, points: Vec<PointStruct>) -> Result<()>`, cùng `UpsertPointsBuilder::new(name, points).wait(true)`, chỉ khác tên collection (`graph_{tenant_id}` vs `chunks_{tenant_id}`). Theo spec T33 — encapsulation, caller không cần biết collection naming convention.
- **`search_graph_nodes` KHÁC `search_chunks` (T32):** Nhận `workspace_id: Uuid` trực tiếp thay vì `Option<Filter>`. Build filter nội bộ: `Filter::must([Condition::matches("workspace_id", workspace_id.to_string())])`. Theo spec T33 — graph search luôn workspace-scoped (không có use case cross-workspace graph query). Caller API gọn hơn (không cần tự build Filter).
- **Filter UUID = keyword match (string) — CONFIRM WORKING lần 2:** T32 đã verify Qdrant Uuid-index (`workspace_id: FieldType::Uuid` từ T29) chấp nhận `Condition::matches("workspace_id", ws.to_string())` → `MatchValue::Keyword`. T33 test GREEN: node 3 (ws_b) bị filter loại đúng, chỉ trả ws_a node. Pattern ổn định, reuse cho mọi filter UUID trong project.
- **Recall test design:** 3 node vector trực giao (`unit_vec(0)`, `unit_vec(1)`, `unit_vec(0)`). Query A=`unit_vec(0)`: cosine(A,A)=1.0 (node 1 + node 3), cosine(A,B)=0.0 (node 2). Filter ws_a giữ node 1 (score 1.0) + node 2 (score 0.0), loại node 3 (ws_b). Top hit = node 1 (entity "Person") → recall đúng. Distinguish node 1 vs node 2 qua `entity_name` (vì chunk_index không có trong graph schema).
- **`with_payload(true)`:** Include payload (`node_id`, `workspace_id`, `entity_name`) trong `ScoredPoint` → caller join `node_id` với `graph_nodes` table PostgreSQL (T21 migration đã có `qdrant_point_id` column pattern) để lấy full node metadata.
- **`upsert_graph_nodes` không check collection tồn tại:** Giống T31 — caller phải `setup_tenant_collections` trước. Upsert vào `graph_{t}` chưa tồn tại → Qdrant error rõ ràng.
