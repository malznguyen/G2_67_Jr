## Quyết định kỹ thuật
- **Reuse `gmrag_core::config::S3Config`** thay vì tạo `WorkerConfig`: `S3Config` đã có đầy đủ fields (`endpoint`, `region`, `access_key`, `secret_key`, `bucket`, `force_path_style`). `S3Client::new(&cfg.s3)` dùng trực tiếp — tránh duplication.
- **`S3Client::new` sync (không async)**: `aws_sdk_s3::Config::builder().build()` là sync operation — không cần async. Tests gọi `S3Client::new(&cfg)` không cần `.await`.
- **Manual config builder (không `aws_config::load_defaults`)**: MinIO cần custom endpoint + static credentials. `aws_config::load_defaults()` auto-discovers từ AWS env vars (AWS_ACCESS_KEY_ID, etc.) — không phù hợp MinIO. Build `Config::builder()` manually với `endpoint_url`, `credentials_provider`, `force_path_style`.
- **`behavior_version_latest()`**: aws-sdk-s3 v1 yêu cầu `behavior_version` được set. Convenience method `.behavior_version_latest()` thay vì `.behavior_version(BehaviorVersion::latest())` — cleaner.
- **`force_path_style(cfg.force_path_style)`**: MinIO yêu cầu path-style addressing (`http://endpoint/bucket/key`) thay vì virtual-host-style (`http://bucket.endpoint/key`). `S3Config.force_path_style = true` trong `.env` cho MinIO.
- **wiremock matching**: Match on `method` + `path` only (ignore query params, Authorization headers, body content). AWS SDK sends SigV4 headers — wiremock ignores since không match trên headers. Path-style URL: `/gmrag-uploads/{key}`.
- **`aws-credential-types` in Cargo.toml**: `Credentials` type re-exported từ `aws_sdk_s3::config::Credentials`, nhưng `aws-credential-types` crate được giữ trong deps cho explicit dependency declaration. Có thể remove sau nếu clippy không báo unused.
