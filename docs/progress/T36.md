pub async fn parse_pdf(data: Vec<u8>, timeout_secs: u64) -> anyhow::Result<ParsedDocument>

## Quyết định kỹ thuật
- **lopdf-first approach (deviation from prompt order):** Prompt đề xuất "thử pdf-extract trước, fallback lopdf". Thực tế: gọi `lopdf::Document::load_mem` đầu tiên để lấy page_count (luôn cần), rồi gọi `pdf_extract::extract_text_from_mem` cho text. Lý do: (1) page_count luôn cần từ lopdf dù pdf-extract thành công hay thất bại; (2) nếu bytes không phải PDF, lopdf fail sớm → return Err ngay, không waste pdf-extract call; (3) tránh load PDF 2 lần trong lopdf (pdf-extract dùng lopdf nội bộ). Trade-off: nếu pdf-extract success, PDF được load 2 lần (1 trong lopdf trực tiếp, 1 trong pdf-extract nội bộ) — acceptable cho worker processing 1 job tại thời điểm.
- **`spawn_blocking` + `tokio::time::timeout`:** pdf-extract và lopdf đều là CPU-bound sync code. Wrap trong `tokio::task::spawn_blocking` để không block async runtime. `tokio::time::timeout(Duration::from_secs(timeout_secs), ...)` → `Err(PdfParseError::Timeout)` nếu quá hạn.
- **Timeout test với `timeout_secs = 0`:** `Duration::from_secs(0)` = `Duration::ZERO`. `tokio::time::timeout(Duration::ZERO, spawn_blocking(...))` poll `JoinHandle` once → `Pending` (blocking thread chưa complete) → timer fires immediately → `Err(Elapsed)`. Reliably triggers timeout mà không cần `tokio::time::pause()`.
  - **Fix (post-commit):** Thêm `#[cfg(test)] std::thread::sleep(Duration::from_millis(100))` đầu `parse_pdf_blocking`. Lý do: `tokio::time::timeout` polls inner future trước — nếu `JoinHandle` `Ready` (blocking thread đã complete với warm pool), timeout không fire. 596-byte PDF parse quá nhanh → race condition. 100ms sleep đảm bảo `JoinHandle` `Pending` khi timeout poll → timeout reliably fires. Test-only, không ảnh hưởng production.
- **PDF fixture generated bằng Python:** Tạo 596-byte valid PDF với Python script — correct xref offsets, 1 page, Helvetica font, text "Hello from GMRAG test". Dùng `include_bytes!("../tests/fixtures/sample.pdf")` trong test. Tránh circular dependency (không dùng lopdf để generate PDF rồi test lopdf).
- **Test assertion linh hoạt:** `assert!(!text.is_empty() || extraction_method == ExtractionMethod::Fallback)` — chấp nhận cả 2 path: pdf-extract extract text thành công (non-empty text) hoặc fallback (text empty nhưng page_count > 0). Đảm bảo test không phụ thuộc vào pdf-extract có handle minimal PDF hay không.
- **`lopdf` version match:** pdf-extract 0.7.12 cũng dùng lopdf 0.34.0 → chỉ 1 version lopdf compiled. Không cần pin lopdf xuống 0.31. Verified qua compile output: chỉ `lopdf v0.34.0` xuất hiện.
