## Quyết định kỹ thuật
- **`PageRenderer` trait thay vì `pdfium-render` trực tiếp**: T37 delivery = OCR orchestration logic (detect thin text → render → OCR → merge). `PageRenderer` trait cho phép test với `MockRenderer` (không cần libpdfium). `PdfiumRenderer` (production, dùng `pdfium-render` + libpdfium trong Docker) là follow-up — trait làm plug-in dễ dàng. Lý do: (1) libpdfium cần download binary cho Windows dev + Docker build stage; (2) OCR orchestration logic là phần custom cần test, pdfium rendering là mechanical wrapper; (3) tránh heavy dep (`pdfium-render` + `image` crate) trong T37 khi logic không phụ thuộc nó.
- **`OcrClient` trait + `MockOcr` + `NoOcr`**: `MockOcr` returns canned text cho OCR-triggered tests. `NoOcr` panics if called — verifies OCR NOT triggered for text PDFs (stricter than MockOcr with distinctive string). Pattern match v1 `vision_ocr_fallback` nhưng THẬT (Ollama vision, không phải stub `mock_ocr_text`).
- **Per-page text extraction via `output_doc_page`**: T36 dùng `extract_text_from_mem` (whole-document). T37 cần per-page để detect thin text per page → dùng `pdf_extract::{PlainTextOutput, output_doc_page}` (port v1 `pdf_parser.rs:74-86`). `extract_page_text(doc, page_number)` wraps `output_doc_page` + error→empty string.
- **`MIN_TEXT_CHARS=50` threshold**: Page với <50 extracted chars → `needs_ocr=true`. Port v1 `MIN_TEXT_CHARS` constant. `text_rich.pdf` fixture có 100 chars → above threshold → OCR not triggered. `scanned.pdf` fixture (blank page) → 0 chars → OCR triggered.
- **`spawn_blocking` + 3-level `Result`**: `tokio::time::timeout(d, spawn_blocking(move || extract_pages_blocking(&data)))` → `Result<Result<Result<Vec<PdfPage>, PdfParseError>, JoinError>, Elapsed>`. Match 3 `Ok` layers: `Ok(Ok(Ok(pages)))` → `Vec<PdfPage>`, `Ok(Ok(Err(e)))` → parse error, `Ok(Err(je))` → thread join error, `Err(_)` → timeout.
- **`data.clone()` cho render**: `parse_pdf_with_ocr` takes `Vec<u8>`. `data` moved vào `spawn_blocking` closure. `data_for_render = data.clone()` trước move → available cho `renderer.render_page_to_png(&data_for_render, page_number)` sau. Clone OK cho typical PDF sizes (<100MB).
- **`ExtractionMethod::Ocr` khi bất kỳ page dùng OCR**: `used_ocr` flag set true nếu bất kỳ page triggers OCR. Final method: `Ocr` if used_ocr, `PdfExtract` if text extracted without OCR, `Fallback` if no text at all.
- **Ollama `/api/chat` format**: Request `{model, messages:[{role:"user", content:OCR_PROMPT, images:[base64]}], stream:false}`. Response `{message:{role:"assistant", content:"extracted text"}}`. `OCR_PROMPT = "Extract all text from this image. Output only the text, nothing else."` — hướng dẫn vision model output raw text.
- **2 fixture PDFs**: `scanned.pdf` (329 bytes, blank page, 0 chars extractable) + `text_rich.pdf` (725 bytes, Helvetica text, 100 chars). T36 `sample.pdf` (596 bytes, 22 chars) quá ngắn cho T37 test (< MIN_TEXT_CHARS=50) → cần `text_rich.pdf` riêng. Cả 3 fixtures generated bằng Python script với correct xref offsets.
