## Quyết định kỹ thuật
- **`new_with_url(host, model)` constructor cho tests**: Production dùng `new(&OllamaConfig)` (reads `cfg.host` + `cfg.embed_model`). Tests dùng `new_with_url(&server.uri(), "nomic-embed-text")` để point tới wiremock. Tránh cần override env vars hay mock `OllamaConfig`. URL = `format!("{}/api/embed", host.trim_end_matches('/'))` — trim trailing slash để không bị `//api/embed`.
- **Builder pattern cho tuning**: `with_batch_size`/`with_concurrency`/`with_timeout_secs`/`with_retries`/`with_backoff_ms` — thay vì read env trực tiếp (v1 `processor.rs:568-586` dùng env-helpers), T39 giữ tuning trong struct fields với defaults. Caller (T42 `process_job` real implementation) có thể đọc env và apply via builders, hoặc dùng defaults. Lý do: testable (không phụ thuộc env), không duplicate env-reading logic vào core `OllamaConfig` (tránh churn core + chạy lại 16 test core). Sau này nếu api crate cũng cần tuning → bump lên `core::OllamaConfig`.
- **`buffer_unordered` + stitch by index**: Batch `(start_index, Vec<String>)` → `buffer_unordered(concurrency)` chạy nhiều batch song song → stitch vào `ordered: Vec<Option<Vec<f32>>>` tại `start + offset`. Output order = input order dù concurrency chạy batch 2 trước batch 1. Test `ollama_embed_batch_multi_batch_preserves_order` verify với 2 mock khác nhau (input `["first"]` → vec768(0.1), input `["second"]` → vec768(0.2)).
- **Retry/backoff port từ v1**: `for attempt in 0..=retries` (tức `retries + 1` total attempts). Backoff `backoff_ms * 2^attempt` capped `2^BACKOFF_CAP_POWER=6` (tức max 64× = 16s với default 250ms). `tokio::time::timeout` per-attempt — nếu timeout, retry tiếp. Last error saved vào `last_error: Option<EmbedError>` → return sau khi exhaust retries. Match v1 `processor.rs:304-337`.
- **Empty input short-circuit**: `if texts.is_empty() { return Ok(Vec::new()); }` trước khi touch network — test `ollama_embed_batch_empty_input_returns_empty` verify không mock mounted vẫn Ok. Tránh wiremock "no matching mock" error.
- **CountMismatch per-batch**: Mỗi batch check `parsed.embeddings.len() != texts.len()` — không phải check total. Test `ollama_embed_batch_count_mismatch` gửi 2 inputs, mock trả 1 embedding → CountMismatch{expected:2, actual:1}.
- **reqwest features match api crate**: `default-features=false` + `json` + `rustls-tls`. Workspace `sqlx` dùng `runtime-tokio-rustls` → giữ TLS backend nhất quán (rustls, không native-tls/openssl). Tránh link 2 TLS stacks.
- **`vec768(seed)` test helper**: `vec![seed; 768]` — mỗi vector có cùng giá trị tại mọi position, distinguished bằng `seed`. Assert `out[0][0] == 0.1` verify order. Đơn giản hơn generate unique vectors.
