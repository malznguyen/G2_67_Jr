    tenant_id   UUID PRIMARY KEY REFERENCES tenants(id) ON DELETE CASCADE,
    provider    TEXT NOT NULL,          -- 'ollama' | 'openai'

## Quyết định kỹ thuật
- **`dimensions=768` pinned cho OpenAI**: `OpenAiEmbedRequest` gửi `{model, input, dimensions: 768}`. `text-embedding-3-small` hỗ trợ shortening (1536→768) → vector khớp `QdrantStore::EMBED_DIM=768` (core/src/qdrant/store.rs:21). KHÔNG cần sửa QdrantStore, KHÔNG cần re-embed khi tenant đổi provider (giải risk R21 "Đang xử lý" → "Đã giải"). Trade-off: mất một nửa thông tin embedding (768 thay vì 1536 native) — acceptable cho MVP, có thể mở rộng sau.
- **`Embedder` trait + boxed future**: `embed_batch` trả `EmbedFuture<'a> = Pin<Box<dyn Future<Output=Result<Vec<Vec<f32>>,EmbedError>> + Send + 'a>>`. `Box::pin` trong impl cho phép `async move` block. `Send` bound để future có thể spawn trên tokio. `'a` tie để borrow `&[String]` không clone. Type alias giải clippy `type_complexity`.
- **`select_embedder` RLS-correct từ đầu**: `BEGIN → SET LOCAL ROLE gmrag_app → SET LOCAL app.tenant_id = '{tenant_id}' → SELECT → COMMIT`. `SET LOCAL ROLE` works từ superuser (gmrag) và no-op từ app_pool (đã gmrag_app qua `after_connect`). Tự chứa — caller chỉ cần truyền pool, không cần setup RLS ngoài. Test `select_embedder_respects_rls_isolation` verify tenant B không thấy config của tenant A.
- **Plaintext api_key (MVP)**: `api_key TEXT NULL` trong `tenant_llm_config`. RLS enforced (FORCE + policy) → chỉ tenant owner thấy key của mình. Follow-up: mã hóa `pgp_sym_encrypt` + env `GMRAG_TENANT_KEY_ENCRYPTION_KEY` (ngoài scope MVP, ghi rõ trong migration comment).
- **Fallback logic 3-level**: (1) không có row → ollama; (2) row `provider='ollama'` → ollama; (3) row `provider='openai'` + `api_key IS NULL` → ollama (can't call OpenAI without key); (4) row `enabled=false` → ollama. Test `select_embedder_openai_without_api_key_falls_back` + `select_embedder_falls_back_when_disabled` verify cases 3+4.
- **`OpenAiEmbedRequest`/`OpenAiEmbedResponse`**: Request `{model, input, dimensions}`. Response `{data:[{embedding, index}]}` — OpenAI format khác Ollama (`embeddings` vs `data[].embedding`). Parse `data.into_iter().map(|d| d.embedding).collect()`.
- **`bearer_auth(&self.api_key)`**: reqwest builder method set header `Authorization: Bearer {key}`. Test `openai_embedder_sends_bearer_auth` verify via `wiremock::matchers::header("Authorization", "Bearer sk-test-key")`.
- **Integration test file riêng**: `tests/select_embedder.rs` (không nằm trong `#[cfg(test)] mod tests` của lib) — vì `#[sqlx::test]` cần DB sống, tách khỏi lib unit tests (wiremock, không cần DB). `migrations = "../../migrations"` relative to worker crate root → `backend/migrations` (match api crate pattern).
- **No `.sqlx` offline cache needed**: `select_embedder` dùng `sqlx::query_as::<_, T>(...)` FUNCTION (runtime SQL), không phải `sqlx::query_as!` MACRO (compile-time checked). Build không cần DATABASE_URL. CI `SQLX_OFFLINE=true` works.
