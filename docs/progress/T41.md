#### Quyết định kỹ thuật
- **Tolerant JSON parser (`parse_graph_json`)**: DeepSeek đôi khi bọc JSON trong ```json … ``` fence hoặc thêm prose dù system prompt ép `response_format: json_object`. Parser: (1) trim; (2) nếu có ``` fence → cắt body giữa fence; (3) locate first `{` và last `}`; (4) deserialize vào `RawExtraction` với `#[serde(default)]` trên `nodes`/`edges` → missing `edges` mặc định `[]`. Test `parse_graph_json_returns_err_on_garbage` verify text không chứa JSON object → `Err(Parse)`.
- **`response_format: {type: "json_object"}`**: Ép DeepSeek/OpenAI trả JSON object thay vì prose. Cả DeepSeek (`deepseek-v4-flash`) và OpenAI (`gpt-4o-mini`) hỗ trợ param này. Parser vẫn tolerant để handle edge cases (model ignore param).
- **System prompt hard-coded `SYSTEM_PROMPT`**: Prompt dict shape `{nodes:[{kind,label,description}], edges:[{source,target,kind}]}`, cấm markdown fence + commentary. `description` dùng để embed graph node (T42) — không bắt buộc (`#[serde(default)]`).
- **Retry/backoff kế thừa T39**: `for attempt in 0..=retries`, backoff `backoff_ms * 2^attempt` capped `2^BACKOFF_CAP_POWER=6` (16s với default 250ms). Per-attempt `tokio::time::timeout`. Test `deepseek_extractor_retries_on_5xx_then_succeeds` (mock 500 ×1 rồi 200) + `deepseek_extractor_fails_after_max_retries` (mock 500 luôn → Err).
- **`select_graph_extractor` 2-layer fallback**: (1) tenant có `tenant_llm_config` enabled + `api_key` + `llm_model` → BYOK extractor pointed at `llm_base_url` (fallback theo `provider`: openai → `api.openai.com/v1`, deepseek → global base_url); (2) tenant có `api_key` nhưng thiếu `llm_model` → fallback global (tránh gọi OpenAI chat với embedding model name); (3) no row / disabled / ollama → global `DeepSeekConfig`. RLS tx `SET LOCAL ROLE gmrag_app` + `SET LOCAL app.tenant_id` (mirror `select_embedder` T40). Test `select_graph_extractor_respects_rls_isolation` verify tenant B không thấy config tenant A.
- **`workspace_id` trên `graph_nodes`**: Migration thêm cột `workspace_id` (NULL cho back-compat) + `UNIQUE(tenant_id, workspace_id, label, kind)` để T42 `ON CONFLICT DO UPDATE` idempotent trên retry. `NULL` workspace_id được phép nhiều row (PostgreSQL unique coi NULL khác nhau) — worker luôn insert non-NULL workspace_id, nên idempotency áp dụng cho future rows.
- **`DeepSeekGraphExtractor` dùng cho cả DeepSeek + OpenAI BYOK**: Cả hai đều OpenAI-compatible `/chat/completions` → cùng struct, chỉ khác `base_url`/`api_key`/`model`. Không cần `OpenAiChatClient` riêng — giảm duplication.
- **`GraphExtractor` trait có `url()`/`model()`**: Cho phép integration test inspect endpoint/model trên `Box<dyn GraphExtractor>` (verify BYOK thật sự trỏ OpenAI, không phải global). Trait object-safe.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* **T42 (dual-write)**: `select_graph_extractor` cung cấp extractor, T42 gọi nó per job rồi `extractor.extract(&full_text)` → `GraphExtraction` → embed `node.description` (dùng `select_embedder` chung với chunks) → `upsert_graph_nodes` Qdrant + `ON CONFLICT (tenant_id, workspace_id, label, kind) DO UPDATE` Postgres. Edges: map `source`/`target` label → `node_id` sau khi insert nodes, insert `graph_edges` với `ON CONFLICT (src_node_id, dst_node_id, kind) DO UPDATE`.
* **`workspace_id` NULL trên graph_nodes cũ**: Migration để NULL cho existing rows. Nếu có dữ liệu seed cũ với NULL workspace_id, unique constraint cho phép duplicate — chỉ ảnh hưởng dữ liệu pre-T41. Worker mới luôn set workspace_id non-NULL.
* **`response_format: json_object` cần model hỗ trợ**: `deepseek-v4-flash` + `gpt-4o-mini` hỗ trợ. Nếu tenant BYOK model không hỗ trợ → API có thể reject hoặc ignore → parser tolerant vẫn handle prose-wrapped JSON, nhưng nếu model trả纯 prose không có `{}` → `Err(Parse)`. Job retry (T43) sẽ catch.
* **OCR/PDF text feed**: T41 `extract(text)` nhận raw text. T43 sẽ truyền `parsed.text` (từ `parse_pdf_with_ocr`) hoặc join chunks. Document rất dài → cần chunk-level extraction hoặc truncate input trong future perf task (ngoài scope T41).
