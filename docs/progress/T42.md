#### Quyết định kỹ thuật
- **Transaction cross-service Postgres ↔ Qdrant**: Qdrant không tham gia ACID Postgres, nên chiến lược: (1) `BEGIN` Postgres tx + upsert metadata; (2) gọi `QdrantStore::upsert_chunks`/`upsert_graph_nodes` **trước** `COMMIT`; (3) nếu Qdrant Err → drop tx (Postgres `ROLLBACK` tự động qua `Drop`); (4) nếu OK → `COMMIT`. Test `dual_write_rolls_back_postgres_when_qdrant_fails` verify: wrong-dim 4-vector → Qdrant reject → 0 rows `document_chunks`/`graph_nodes`. Trade-off: nếu Postgres `COMMIT` fail sau Qdrant success (rare) → orphan Qdrant points; retry re-upsert cùng point id (idempotent) → acceptable cho MVP.
- **Idempotency `qdrant_point_id` stable trên retry**: `document_chunks` INSERT dùng `gen_random_uuid()` cho `qdrant_point_id`, nhưng `ON CONFLICT (document_id, chunk_index) DO UPDATE SET content = EXCLUDED.content` **KHÔNG** ghi đè `qdrant_point_id` → `RETURNING qdrant_point_id` trả về UUID gốc trên retry. Test `dual_write_preserves_idempotency_on_retry` verify `res1.chunk_ids == res2.chunk_ids`. Nếu ghi đè `qdrant_point_id = EXCLUDED.qdrant_point_id` (random mới) → Qdrant point mới mỗi retry → leak points; fix bằng không update cột đó.
- **`graph_nodes` idempotent qua `UNIQUE(tenant_id, workspace_id, label, kind)`**: Migration T41 thêm constraint. `ON CONFLICT ... DO UPDATE SET properties = EXCLUDED.properties` → `RETURNING id` stable. Map `label → id` (HashMap) để edges lookup. Nếu extraction có 2 node cùng label/kind → `or_insert` giữ id đầu → edge trỏ đúng node. Test verify `res1.node_ids == res2.node_ids`.
- **`graph_edges` idempotent qua `UNIQUE(src_node_id, dst_node_id, kind)`** (schema T21 sẵn có): `ON CONFLICT DO UPDATE SET weight = EXCLUDED.weight`. Edges có endpoint label không nằm trong extraction → skip (continue) + không count. Test `edges_written == 1` cho 1 edge hợp lệ.
- **Qdrant point id = UUID string**: `qdrant_client::PointId` có `From<String>` → `PointIdOptions::Uuid(String)`. Dùng `point_id.to_string()` / `node_id.to_string()` làm point id. `Uuid` không impl `Into<PointId>` trực tiếp (verify qdrant-client 1.12.1 `primitives.rs:181`) → phải qua string. Qdrant accept UUID-form string làm point id.
- **`setup_tenant_collections` trước upsert**: Gọi idempotent setup trong tx (Qdrant op, không ảnh hưởng Postgres tx). Đảm bảo `chunks_{tenant_id}`/`graph_{tenant_id}` tồn tại trước upsert — tránh error "collection doesn't exist". T30 setup đã guard `collection_exists`.
- **Payload schema khớp T28/T29**: chunk payload `{workspace_id, document_id, chunk_index, filename, owner_id, visibility}` (T28 indexes); node payload `{node_id, workspace_id, entity_name}` (T29 indexes). UUID lưu string (Qdrant payload index `Uuid` accept string).
- **`IngestError` enum**: `Db(String)` (sqlx), `Qdrant(String)` (gmrag_core::Error), `Input(String)` (length mismatch). `From<sqlx::Error>` + `From<gmrag_core::Error>` cho `?` ergonomics. Qdrant `Payload::try_from` error → `Qdrant(String)`.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* **T43 (`process_job` hoàn chỉnh)**: T42 cung cấp `dual_write_ingestion` nhưng chưa wire vào `process_job`. T43 phải: (1) khởi tạo `app_pool` (`init_app_pool`) + `QdrantStore` + `S3Client` trong `run()`; (2) per job: `SET LOCAL app.tenant_id`, download S3 → `parse_pdf_with_ocr` → `chunk_page_texts` → `select_embedder` → `embed_batch(chunks)` + `embed_batch(node descriptions)` → `select_graph_extractor` → `extract(full_text)` → `dual_write_ingestion`; (3) cập nhật `ingest_jobs.status` (processing/completed/failed) + `attempts` + `last_error`; (4) in-memory backoff retry (max 3, cap 16s).
* **`IngestJob` cần `owner_id` + `visibility`**: T42 `DualWriteInput` nhận `owner_id`/`visibility`. Struct `IngestJob` hiện thiếu 2 field này → T43 phải bổ sung vào struct + serde (API enqueue sẽ populate).
* **`documents.status` update**: T42 chưa update `documents.status` ('processing'/'indexed'). T43 nên update trong `process_job` wrapper (hoặc thêm vào `dual_write_ingestion` nếu muốn). Hiện T42 chỉ ghi `document_chunks`/`graph_nodes`/`graph_edges`.
* **Graph node embed dùng `node.description`**: T43 phải embed `extraction.nodes[i].description` (không phải `label`) để vector mang ngữ nghĩa mô tả. Dùng cùng `select_embedder` (Ollama/OpenAI 768-dim) cho cả chunks + nodes — đảm bảo cùng semantic space cho hybrid retrieval.
* **Long document → graph extract input**: T41 `extract(text)` nhận raw text. T43 cần quyết định truyền full text hay join chunks. Document rất dài có thể vượt context window DeepSeek → cần truncate hoặc chunk-level extraction trong future perf task.
