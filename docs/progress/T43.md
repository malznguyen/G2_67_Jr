#### Quyết định kỹ thuật
- **`JobRunner` trait cho testability**: Pipeline thật (`IngestContext::process_job`) cần S3 + Qdrant + Ollama + DeepSeek live → không unit-test được. `JobRunner` trait (`run(&self, job) -> Result<(), String>`) cho phép `process_job_with_retry` test với `MockRunner` (fail N lần rồi Ok) + `AlwaysFailRunner` — verify retry/status logic độc lập với pipeline. `IngestContext` impl `JobRunner` với `run = process_job`. Pattern giống `JobQueue`/`MockQueue` (T34).
- **Retry in-memory, không re-enqueue Redis**: `process_job_with_retry` loop `0..MAX_ATTEMPTS` với `tokio::time::sleep` giữa các fail. Tuyệt đối không `LPUSH` lại Redis (tránh spin-loop ngốn CPU per yêu cầu). Sau 3 fail → `status='failed'`, job drop khỏi worker context (Redis BRPOP đã consume). API có thể re-enqueue manual sau nếu muốn.
- **Backoff `1s * 2^attempt` cap 16s**: `BACKOFF_BASE_MS=1000`, `BACKOFF_CAP_MS=16000` (= T39's `250ms * 2^6`). Attempt 0 fail → sleep 1s, attempt 1 fail → sleep 2s, attempt 2 fail → mark failed (no sleep). Test `retry_marks_failed_after_three_attempts` pay real backoff (~3s) — acceptable cho CI.
- **`process_job_with_retry` luôn trả `Ok(())`**: Trả `Err` chỉ khi `update_job_status` (DB) fail. Job pipeline error → recorded trong `ingest_jobs.last_error` + status, wrapper trả `Ok` → poll loop `run()` không crash. Test `.expect("wrapper must not propagate job error")` verify. Chỉ DB error (rare) mới propagate → `tracing::error!` trong loop.
- **`update_job_status` RLS-scoped tx**: `BEGIN` + `SET LOCAL ROLE gmrag_app` + `SET LOCAL app.tenant_id = {tenant_id}` + `UPDATE ingest_jobs`. `init_app_pool` after_connect đã `SET ROLE gmrag_app`, nhưng `SET LOCAL ROLE` redundant-safe (no-op từ app_pool, works từ superuser test pool). `SET LOCAL app.tenant_id` scope RLS policy `ingest_jobs_isolation` (T25) → chỉ update job thuộc tenant hiện tại.
- **`IngestJob` +`owner_id`/`visibility` (required)**: Per T43 design decision — worker stateless, API enqueue nhồi metadata vào payload để worker tránh `SELECT documents` extra. Legacy payload thiếu 2 field → `serde_json` reject (`ingest_job_deserializes_legacy_without_owner_visibility_fails` verify). API route (future) phải populate đầy đủ.
- **Pipeline dùng `parse_pdf` (text path), chưa `parse_pdf_with_ocr`**: T37 `PdfiumRenderer` chưa implement (cần `pdfium-render` + libpdfium binary — blocker T37). T43 dùng `parse_pdf` (T36, text-only) + note OCR wiring là follow-up. Pipeline structure đã hỗ trợ OCR sau (swap `parse_pdf` → `parse_pdf_with_ocr` + `PdfiumRenderer` khi sẵn sàng). Graph extract feed `full_text` (clone trước move vào `page_texts` để tránh borrow-after-move).
- **Graph node embed dùng `description` (fallback `label`)**: `node_texts = nodes.iter().map(|n| if description empty { label } else { description })`. Dùng cùng `select_embedder` (Ollama/OpenAI 768-dim) cho cả chunks + nodes → shared semantic space cho hybrid retrieval. Empty `node_texts` → skip embed (`Vec::new()`), `dual_write_ingestion` handle empty.
- **`IngestContext::from_config`**: Khởi tạo `init_app_pool` (RLS) + `QdrantStore::new` (health probe) + `S3Client::new` (sync). Fail-fast nếu DB/Qdrant unreachable. `run()` gọi `.context("building ingest context")` → clean error. `ctx.pool` clone (PgPool cheap clone via Arc).

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* **OCR pipeline chưa wire**: T43 dùng `parse_pdf` (text-only). Scanned/image-only PDF → empty text → empty chunks → empty graph. Cần implement `PdfiumRenderer` (`pdfium-render` + libpdfium trong Docker) rồi swap sang `parse_pdf_with_ocr(bytes, 30, &renderer, &ocr)` — follow-up task (T37 blocker).
* **API enqueue route chưa có**: `routes/documents.rs` chưa tồn tại. API phải tạo route enqueue `IngestJob` với đầy đủ `owner_id`/`visibility` + `LPUSH gmrag:ingest_jobs`. Worker đã sẵn sàng consume.
* **`documents.status` chưa update**: T43 update `ingest_jobs.status` nhưng chưa update `documents.status` ('processing'/'indexed'/'failed'). Có thể thêm `UPDATE documents SET status` trong `process_job` hoặc `dual_write_ingestion` — follow-up.
* **Long document → DeepSeek context window**: `extract(&full_text)` truyền toàn bộ text. Document rất dài có thể vượt DeepSeek context limit → cần truncate hoặc chunk-level extraction + merge graph — future perf task.
* **Backoff test slow**: `retry_marks_failed_after_three_attempts` pay real backoff 3s. Nếu CI chặt chẽ, có thể inject backoff duration qua trait/config — future refactor.
* **Workspace clippy**: Chỉ chạy `-p gmrag-worker`. Core/api không chạm T43 → nên clean, nhưng chạy `cargo clippy --workspace --all-targets -D warnings` (SQLX_OFFLINE=true) ở cuối sprint để confirm.
