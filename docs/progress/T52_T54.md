#### Quyết định kỹ thuật
- **`/tenants` GET/POST dùng `AdminPool`**: đây là thao tác pre-tenant/cross-tenant (chưa có `X-Tenant-Id`). Bảng `tenants` có FORCE RLS với `WITH CHECK (id = gmrag_current_tenant())`, nên INSERT tenant mới chỉ thành công qua connection bypass RLS.
- **`/tenants/{tid}` + members tenant-scoped**: bắt buộc `SharedConnection` (bất biến RLS Blocker 3 BATCH2B). Guard `tid == TenantContext.0` để không thao tác nhầm tenant khác.
- **Invite = invitations table**: tạo bản ghi `pending` (email + token + status), KHÔNG thêm thẳng vào `tenant_members`; membership tạo sau khi accept (flow accept ngoài scope Sprint 7A).
- **Authz owner-only cho mutation**: `require_owner` đọc role qua RLS-scoped connection; mọi thành viên đều được list.
- **Vocabulary role**: dùng lowercase `owner`/`member` khớp comment migration T12 và test hiện có. Hệ thống phân cấp OWNER>ADMIN>EDITOR>VIEWER sẽ được chuẩn hoá ở T64/T65.
- **Runtime query thay vì macro `query!`**: dùng `sqlx::query*().bind()` (như `auth/tenant.rs` + integration tests) → KHÔNG cần `cargo sqlx prepare`/regen `.sqlx` cache, build hermetic trên host Windows.
- **Không đổi schema/migration**: bảng `tenants`, `tenant_members`, `invitations`, RLS policies và GRANT cho `gmrag_app` đã tồn tại từ T12/T19/T23/T25.
