#### Quyết định kỹ thuật
Cách viết SQL cho filter visibility + ACL và đảm bảo RLS:

```sql
SELECT id, title, visibility, owner_id, workspace_id, status, created_at
FROM documents
WHERE (
        visibility = $1                         -- $1 = 'shared'
        OR owner_id = $2                         -- $2 = current_user
        OR (workspace_id IS NOT NULL AND EXISTS (
              SELECT 1 FROM workspace_members wm
              WHERE wm.workspace_id = documents.workspace_id
                AND wm.user_id = $2))
      )
  AND ($3::uuid IS NULL OR workspace_id = $3)     -- $3 = optional filter
ORDER BY created_at DESC
```

- **2 lớp bảo vệ**: RLS (lớp DB bắt buộc) cô lập theo `tenant_id = gmrag_current_tenant()` trước; biểu thức `WHERE` chỉ thêm visibility/ACL theo user. Subquery `workspace_members` cũng chạy trong cùng RLS context nên không lộ tenant khác. Test `documents_are_tenant_isolated` xác nhận doc `shared` của tenant B không lọt vào listing tenant A.
- **"public" = `visibility = 'shared'`**: theo đúng mệnh đề SQL cụ thể trong spec (không phải literal `'public'`). Schema không có CHECK constraint trên `visibility` nên giá trị là quy ước ở tầng app; hằng số `VISIBILITY_SHARED` để dễ đổi.
- **Phạm vi ACL**: chỉ dựa trên `workspace_members` (theo mệnh đề `EXISTS` trong spec). Bảng `resource_acl` (chia sẻ theo principal cụ thể) KHÔNG nằm trong endpoint list này — để dành cho task ACL chi tiết sau.
- **Filter `workspace_id`**: nhận qua `Query<DocListParams>`; cast `$3::uuid` để nhánh `IS NULL` short-circuit khi không truyền.
- **URL Guard + SharedConnection**: `ensure_path_matches_context(tid, &ctx)` đầu handler; chạy query trên `SharedConnection` (bất biến RLS Blocker 3).
- **Runtime query** (không macro `query!`) → không cần `cargo sqlx prepare`.
