#### Quyết định kỹ thuật
- **All-or-Nothing rollback S3 dưới mô hình SharedConnection**: `rls_middleware` mở 1 transaction cho cả request và **COMMIT vô điều kiện sau khi handler trả về (kể cả khi handler trả lỗi)**. Vì vậy handler tự đảm bảo nguyên tử: (1) S3 `put` trước; (2) `SAVEPOINT sp_upload`; (3) INSERT `documents` (status=`uploaded`) + INSERT `ingest_jobs` (status=`pending`) RETURNING id; (4) redis LPUSH; (5) nếu BẤT KỲ bước DB/queue nào fail → `ROLLBACK TO SAVEPOINT sp_upload` (undo 2 INSERT trong outer tx) **và** `store.delete(s3_key)` (dọn file rác) rồi trả lỗi; (6) thành công → `RELEASE SAVEPOINT`. Test `upload_rolls_back_s3_and_db_when_enqueue_fails` ép Redis fail, verify S3 delete được gọi đúng key và 0 row `documents`/`ingest_jobs`.
- **Trait + mock cho S3/Redis (không cần infra live trong test)**: theo đúng pattern `JobQueue`/`JobRunner` của worker (T34/T43). Handler nhận `Extension<Arc<dyn ObjectStore>>` + `Extension<Arc<dyn JobEnqueuer>>`; test inject `MockObjectStore`/`MockEnqueuer` ghi lại call + ép lỗi. Test chạy handler qua `Router` + `oneshot` (multipart cần body thật) với `SharedConnection` dựng từ helper `rls_conn` (BEGIN + SET ROLE gmrag_app + SET app.tenant_id) như T57.
- **`IngestJobPayload` khớp worker `IngestJob`**: nhồi sẵn `owner_id` + `visibility` + `workspace_id` (T43 yêu cầu, worker reject payload legacy thiếu field). `payload.id == ingest_jobs.id` để `update_job_status` của worker target đúng row.
- **Validation visibility qua Serde**: `Visibility { Shared, Private }` (`rename_all="lowercase"`), parse bằng `serde_json::from_value` → giá trị ngoài `shared`/`private` (vd `public`) → 400 trước khi chạm DB/S3.
- **Quota check**: đọc `tenant_quotas (max_storage_bytes, max_documents)` qua SharedConnection (RLS); so `SUM(byte_size)+new` và `COUNT+1`. Không có row quota → không enforce (mặc định mở). Vượt → `QuotaExceeded` 429, chặn trước khi S3 put.
- **S3 key convention**: `{tenant_id}/{workspace_id}/{document_id}.pdf` (theo spreadsheet). `workspace_id` bắt buộc ở upload (worker `IngestJob.workspace_id` non-optional).
- **Body limit**: `DefaultBodyLimit::max(50 MiB)` cho nhóm tenant-scoped (mặc định axum chỉ 2 MiB).

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* **Race COMMIT cuối của middleware**: redis LPUSH chạy trong handler TRƯỚC khi `rls_middleware` COMMIT outer tx. Nếu LPUSH thành công nhưng COMMIT cuối fail (hiếm), job đã vào queue nhưng `documents`/`ingest_jobs` bị rollback → worker sẽ fail (FK `document_chunks` → `documents` không tồn tại) và mark job `failed` sau retry. Không thể commit giữa handler vì middleware sở hữu tx. Chấp nhận degrade; cân nhắc outbox pattern nếu cần đảm bảo tuyệt đối.
* **`documents.status` lifecycle**: T58 set `uploaded`; worker (T43) hiện chỉ update `ingest_jobs.status`, chưa update `documents.status` (`processing`/`indexed`/`failed`) — follow-up.
* **`delete_chunks_by_document` mới ở core**: T59 dùng. Chỉ áp cho `chunks_{tenant}` (graph nodes KHÔNG có document link → không dọn per-document, xem T59).
