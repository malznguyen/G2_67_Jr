Thứ tự: **S3 delete → Qdrant chunk delete → Postgres DELETE** (theo spec).
- **KHÔNG dọn graph nodes per-document (giới hạn đã biết)**: `graph_nodes`/`graph_edges` + graph Qdrant points (`graph_{tenant}`) KHÔNG có liên kết `document_id`; node được dedupe theo `(tenant, workspace, label, kind)` và dùng chung giữa nhiều document (xem `dual_write_ingestion` T42). Vì vậy không có tập point per-document để xóa an toàn — Postgres cascade cũng không đụng `graph_nodes`. Quyết định (đã xác nhận với chủ dự án): cleanup chỉ áp dụng cho chunks; graph nodes giữ nguyên. Muốn dọn graph thật sự cần migration thêm liên kết document→node (task tương lai).
- **Owner-only guard**: SELECT `s3_key, owner_id` trước; `owner_id != current_user` → 403; cross-tenant bị RLS ẩn → SELECT rỗng → 404 (phân biệt rõ 403 vs 404). Verify 404 cho cross-tenant + 403 cho non-owner.
