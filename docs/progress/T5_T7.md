#### Quyết định kỹ thuật

1. **Workspace layout** — `core` (lib) + `api` (bin) + `worker` (bin). `core` re-export `Config/Error/init_pool` nên api & worker dùng chung một boot sequence; chỉ tốn 1 dòng `gmrag_core::init_pool` cho mỗi binary.
2. **`sqlx::migrate!("../../migrations")`** — chọn macro compile-time (resolve theo `CARGO_MANIFEST_DIR`) thay vì `Migrator::new("./migrations")` (resolve theo CWD lúc runtime). Macro tránh được lỗi "relative path" tùy thuộc thư mục chạy `cargo run` và đảm bảo Dockerfile + dev local đều trỏ về cùng `backend/migrations/`.
3. **`/health` + `/healthz`** — task chỉ yêu cầu `GET /health`. Mình add thêm `GET /healthz` làm alias vì `infra/docker-compose.yml` đã commit sẵn healthcheck `wget -qO- http://localhost:8080/healthz` (xem `docs/progress/docker-compose-config.txt:53`). Làm sẵn tránh phải sửa compose trong task T8 và giữ cho `docker compose up` chạy thẳng khi T5 unblock. **Không phải scope creep** — đây là dependency đã có trong T1-T4.
4. **Liveness vs readiness split** — `/health` không touch DB (liveness — orchestrator restart pod nếu fail); `/healthz` ping DB (readiness — pod join service khi DB sẵn sàng). Pattern này chuẩn K8s và free vì test `health_returns_200` dùng được pool lazy (`connect_lazy`).
5. **Config mặc định an toàn** — chỉ `DATABASE_URL` là required (fail fast); `GMRAG_HTTP_BIND`, `GMRAG_RUST_LOG`, `GMRAG_TENANT_HEADER`, `GMRAG_SERVICE_NAME` có default hợp lý. `GMRAG_DEFAULT_TENANT` trong `.env.example` được CỐ TÌNH bỏ qua: invariant #1 của repo là tenant identity phải tới từ request context, không phải config tĩnh. Field này sẽ được wire vào TenantContext middleware ở task sau.
6. **Env test serialised bằng Mutex** — hai test config cùng mutate `DATABASE_URL`/`GMRAG_HTTP_BIND` process-wide. `cargo test` chạy parallel theo mặc định → race. Gộp thành một `config_env_matrix` với `static ENV_LOCK: Mutex<()>` để giữ determinism. Đây là idiom chuẩn thay vì kéo thêm dep `serial_test`.
7. **Multi-stage Dockerfile** — `cargo-chef` để cache dependency layer (lần build sau chỉ rebuild crates có đổi), runtime image `debian:bookworm-slim` + `tini` + non-root user `gmrag` (UID 1001). Build cả `gmrag-api` & `gmrag-worker` rồi để compose override `command:` cho worker. Healthcheck bind `0.0.0.0:8080` match compose expose.
8. **Commit `Cargo.lock`** — convention cho binary workspace (cargo mặc định gitignore `Cargo.lock` cho lib-only workspace, nhưng workspace có bin sẽ commit để CI build reproducible).
9. **Migrations root `backend/migrations/`** — chia sẻ giữa `api` và `worker`, future job crate có thể dùng lại mà không cần symlink. Hiện tại chỉ có 1 placeholder (`20260101000000_init.sql`) — domain tables + RLS thuộc task T8+.

#### Blocker / Warning cho task tiếp theo

* **T8+ (frontend / config TDD refine)**: `Config` hiện chỉ đọc env — KHÔNG có struct tương ứng với mọi key trong `.env.example` (S3, Qdrant, DeepSeek, Keycloak, Ollama, CORS, ...). Task T8 / T9 sẽ bổ sung từng nhóm theo phương pháp TDD: thêm field → viết test mảng-khóa → implement. Cấu trúc crate `gmrag-core` đã đủ chỗ (`config.rs` có thể split thành submodule `config/{database,server,logging,oidc,llm,storage,...}.rs` khi cần).
* **`GMRAG_DEFAULT_TENANT`** trong `.env.example` chưa được dùng. Bất kỳ task nào muốn seed dev data PHẢI tôn trọng invariant #1: tenant identity đi qua `TenantContext` middleware, KHÔNG pull từ URL. Field này sẽ được expose ở `gmrag_core::config` chỉ khi có use-case seeding rõ ràng.
* **Pre-existing `_sqlx_migrations` trong dev Postgres** — local Postgres `gmrag-postgres-1` đang chứa 7 migration records từ một project trước (chat history, initial schema, ...). Mình đã `DROP TABLE _sqlx_migrations` để test xanh, nhưng **T8+ cần chạy `docker compose up -d --force-recreate postgres16` trên volume sạch** để tránh tương tác với schema cũ. T1-T4 đã commit sẵn `infra/postgres/init.sql` đúng spec — chỉ cần volume mới là đủ.
* **pg_hba: cần trust rule** — mình đã thêm `host all all 0.0.0.0/0 trust` vào `pg_hba.conf` của container dev hiện tại để test xanh. Khi `docker compose up` lại từ đầu, image `postgres:16-alpine` mount `/docker-entrypoint-initdb.d/01-init.sql` chỉ chạy **một lần duy nhất** lần đầu tạo volume — pg_hba bên trong image mặc định `127.0.0.1/32 trust`, OK cho dev local. **KHÔNG** ship pg_hba relax này vào init.sql của production.
* **`gmrag-net` 8088→8080 port mapping** — compose map `8088:8080` cho backend. T8+ test bằng `curl http://localhost:8088/health` (không phải `8080`) khi chạy qua compose stack.
* **`backend.Dockerfile` CHƯA được build/test thực tế** trong task này (docker Desktop Windows + WSL2 không có sẵn Rust toolchain trong build context mount). Task T8+ nên `docker compose build backend` để verify build thành công trước khi rely on compose healthcheck.
