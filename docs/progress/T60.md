#### Quyết định kỹ thuật
- **ACL giống `list_documents` (T57)**: trả tài liệu iff `visibility='shared'` HOẶC `owner_id = current_user` HOẶC caller là member của workspace tài liệu (`EXISTS workspace_members`). RLS cô lập tenant trước → cross-tenant và không-đủ-quyền đều ra `404` (không lộ thông tin tồn tại). `fetch_optional().ok_or(NotFound)`.
- **Chunks**: `SELECT chunk_index, content, token_count FROM document_chunks WHERE document_id=$1 ORDER BY chunk_index ASC LIMIT 50`. Test chèn chunk lệch thứ tự (2,0,1) rồi verify trả về `[0,1,2]` để chứng minh `ORDER BY`.
- **Response shape**: `{ "document": { id, title, status, visibility, owner_id, workspace_id, mime_type, byte_size, created_at }, "chunks": [ { chunk_index, content, token_count } ] }`. Dùng `sqlx::FromRow` + `Serialize` cho cả hai struct.
- **2 lớp bảo vệ**: RLS (DB) + biểu thức ACL (app), chạy trên `SharedConnection` đúng bất biến RLS (Blocker 3) + `ensure_path_matches_context` đầu handler.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* **Chunks rỗng khi chưa ingest xong**: preview ngay sau upload sẽ trả `chunks: []` cho tới khi worker hoàn tất pipeline (document `status` vẫn `uploaded`/`processing`). Frontend nên hiển thị trạng thái theo `document.status`.
* **Giới hạn 50 chunk + chưa phân trang**: tài liệu lớn chỉ xem được 50 chunk đầu; cần thêm pagination (offset/cursor) nếu UI cần xem toàn bộ.
* **`resource_acl` chưa áp vào preview**: như T57, ACL hiện chỉ dựa `workspace_members` + owner + shared; chia sẻ theo principal cụ thể (bảng `resource_acl`) là task ACL riêng.
