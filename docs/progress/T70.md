#### Env vars / Config mới thêm
* `AUTH_SECRET=` (server-only, generate `openssl rand -base64 32`)
* `AUTH_TRUST_HOST=true`
* `KEYCLOAK_FRONTEND_CLIENT_SECRET=` (server-only; để trống nếu Keycloak client `gmrag-frontend` là public/PKCE)

#### Quyết định kỹ thuật
* **NextAuth v5**: dùng `NextAuth({...})` trả `{ handlers, signIn, signOut, auth }`. `providers` không phải property của result (khác docs cũ) — provider định nghĩa inline trong config.
* **Keycloak OIDC**: issuer = `${NEXT_PUBLIC_KEYCLOAK_URL}/realms/${NEXT_PUBLIC_KEYCLOAK_REALM}`, `wellKnown` = issuer + `/.well-known/openid-configuration`, scope `openid email profile`. `idToken: true`.
* **access_token**: lưu vào session qua callback `jwt` → `session` để T71 middleware đính `Authorization: Bearer`.
* **shadcn/ui**: viết tay tối thiểu (Button + utils + CSS vars) thay vì `shadcn init` tương tác — giữ TDD deterministic.
* **QueryClient**: tạo trong `useState` (tránh recreate mỗi render), `staleTime: 60s`, `retry: 1`.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* `AUTH_SECRET` và `KEYCLOAK_FRONTEND_CLIENT_SECRET` cần giá trị thật trong `.env` trước khi chạy dev. Nếu Keycloak client `gmrag-frontend` confidential thì phải set secret; nếu public (PKCE) thì để trống — cần xác nhận với DevOps/Keycloak admin.
* `session.access_token` chỉ có sau khi user đăng nhập; T71 client middleware phải handle trường hợp chưa có token (request sẽ 401, cần redirect login).
