#### Quyết định kỹ thuật
* **openapi-fetch 0.17**: middleware API dùng `onRequest`/`onResponse` (KHÔNG phải `middleware`/`onError`). 2 middleware: `authHeadersMiddleware` (gắn `Authorization` + `X-Tenant-ID`) và `errorMiddleware` (throw `ApiError` khi non-2xx).
* **Token tách server/client**: `auth-token.server.ts` (`import "server-only"`, gọi `auth()`) và `auth-token.client.ts` (module-scoped holder, set từ `SessionTokenBridge`). Giữ `@/auth` ra khỏi client bundle → tránh lỗi `next-auth` server-only trong client.
* **Tenant resolver pluggable**: `tenant-resolver.ts` chỉ giữ 1 callback; T72 store sẽ gọi `setActiveTenantResolver(() => useTenantStore.getState().activeTenantId)`. Tránh circular import `client ↔ store`.
* **schema.d.ts fallback**: viết tay tối thiểu từ OpenAPI spec thật (`backend/crates/api/src/openapi/schemas.rs`) để build không phụ thuộc backend online. Script `gen:api` tái sinh khi backend chạy: `openapi-typescript $NEXT_PUBLIC_API_BASE_URL/openapi.json -o lib/api/schema.d.ts`.
* **`GET /users/me` shape**: theo OpenAPI spec `{ user, tenants: [{id,name,role}] }` (KHÔNG flat như contract doc).
* **Tenant header** gửi cho mọi request (kể cả `/users/me`, `/tenants`); backend bỏ qua trên endpoint không tenant-scoped — giữ đơn giản.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* `schema.d.ts` là fallback thủ công; khi backend online cần chạy `pnpm gen:api` để sinh đầy đủ tất cả endpoint (graph SSE, settings, metering, ACL...). Có thể lệch type nếu backend thay schema.
* Middleware `onResponse` throw `ApiError` — caller (`bootstrap` T72, React Query) phải `try/catch` hoặc để error propagate.
* Token client-side chỉ available sau khi `SessionTokenBridge` effect chạy; request trước đó sẽ không có `Authorization` → 401. UI cần gate theo `status === "authenticated"`.
