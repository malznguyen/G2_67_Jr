#### Quyết định kỹ thuật
* **Zustand 5 + persist**: store `tenants[]`, `activeTenantId`, `bootstrapStatus` (`idle|loading|done|error`), `bootstrapError`. `partialize` chỉ persist `activeTenantId` (KHÔNG persist `tenants` — bootstrap lại mỗi session để đồng bộ membership mới).
* **bootstrap()**: gọi `client.GET("/users/me")` (T71) → đọc `data.tenants` (shape OpenAPI `{user, tenants[]}`) → `setTenants`; nếu `activeTenantId === null && tenants.length > 0` → auto-select `tenants[0].id`. Guard `bootstrapStatus === "loading"` chống gọi trùng.
* **Resolver wiring**: tại module load, gọi `setActiveTenantResolver(() => useTenantStore.getState().activeTenantId)` → openapi-fetch middleware (T71) tự gắn `X-Tenant-ID`. Tránh circular import: store import `client` + `tenant-resolver`; `client` import `tenant-resolver` (không import store).
* **TenantBootstrapper**: gate theo `useSession().status === "authenticated"` để token client đã sẵn sàng (T71 `SessionTokenBridge` chạy trước). Chỉ trigger khi `bootstrapStatus === "idle"`.
* **TenantSwitcher**: ẩn khi `≤1` tenant (không cần chọn). Hiển thị `name (role)`.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* **Hydration / persist**: zustand `persist` đọc localStorage ở client; SSR sẽ thấy `activeTenantId = null` rồi hydrate → có thể flash. UI feature sau nên đọc `activeTenantId` trong `useEffect`/client-only hoặc chờ `bootstrapStatus === "done"`.
* **bootstrap chỉ chạy 1 lần/session** (gate `idle`); nếu user đổi tenant hoặc cần refresh membership phải gọi `bootstrap()` thủ công hoặc thêm action `resetBootstrap`.
* **Tenant header gửi cho mọi request** kể cả `/users/me`, `/tenants` — backend bỏ qua, nhưng nếu sau này backend strict trên endpoint non-tenant-scoped thì cần loại trừ path trong middleware.
* `tenants[]` rỗng (user mới chưa có tenant) → `activeTenantId` vẫn `null` → mọi call tenant-scoped sẽ 401/400. UI cần handle empty-tenant state (prompt tạo tenant) ở batch feature tiếp theo.
