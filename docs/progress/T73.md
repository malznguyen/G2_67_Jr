#### Quyết định kỹ thuật
* **next-intl 4.13.0 + `[locale]` segment**: `localePrefix: "as-needed"` → URL mặc định không có prefix cho `vi` (default), `en` có prefix `/en`. Root `app/page.tsx` `redirect("/vi")`, root `app/layout.tsx` là pass-through (không `<html>`) để `[locale]/layout` render `<html lang>`.
* **i18n dictionary pattern**: 2 layer — `request.ts` (server, dynamic import cho `getRequestConfig`) và `dictionary.ts` (import JSON tĩnh, dùng cho `generateStaticParams`/test/server layout truyền messages vào `NextIntlClientProvider`). Tránh ESM dynamic import trên Windows.
* **Theme Zinc minimalist**: CSS variables HSL tông `240` (Zinc) cho cả light (`--background: 0 0% 100%`) và dark (`--background: 240 10% 3.9%`). `--radius: 0.5rem`. Thêm token `card/popover/destructive` cho shadcn sau. Dark mode toggle class `.dark` trên `<html>` (T77 sẽ thêm ThemeToggle).
* **Middleware compose**: `intlMiddleware` chạy trước (xử lý locale), sau đó check `PROTECTED_PATTERN` `/^\/(?:[a-z]{2}\/)?t\/.../` → `auth()` (server, NextAuth v5) → redirect `[locale]/login` khi không session. Matcher loại trừ `api|_next|_vercel|auth|.*\..*`.
* **Path alias `@/*`**: tsconfig `paths` multi-entry `src/*`, `components/*`, `lib/*`, `./*` để vừa hỗ trợ module mới trong `src/` vừa giữ import cũ `@/lib/api/client`, `@/components/providers` (T70-T72, T84) không phải refactor.
* **AppShell**: client component (cần `useSession` cho nút Sign out). TenantSwitcher (T72) nhúng trong header; ẩn tự động khi ≤1 tenant.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* `auth()` trong middleware chạy trên Node runtime (mặc định), không phải edge thuần — đảm bảo `AUTH_SECRET` có ở runtime. Nếu sau này chuyển `runtime: "edge"` cần verify Keycloak OIDC provider tương thích.
* `localePrefix: "as-needed"` khiến URL `vi` không có prefix; khi tạo link nội bộ LUÔN dùng `Link`/`useRouter` từ `@/i18n/navigation` (không `next/link` gốc) để giữ locale context.
* Root `app/layout.tsx` là pass-through (không `<html>`/`<body>`) — mọi metadata/lang nằm ở `[locale]/layout`. Không được thêm `<html>` thứ hai.
* Tenant layout `/t/[tid]` KHÔNG tự validate `tid` thuộc user (chỉ auth gate). UI feature (T75 documents) phải đọc `activeTenantId` từ Zustand store và handle mismatch `tid` URL vs active tenant.
* `setRequestLocale` phải gọi trong MỖI layout/page con để SSG static rendering per-locale hoạt động — hiện chỉ gọi ở `[locale]/layout`, `[locale]/t/[tid]/layout`, `[locale]/t/[tid]/w/[wid]/layout`. Nếu thêm page sâu hơn, gọi lại `setRequestLocale(locale)`.
