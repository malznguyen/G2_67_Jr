#### Quyết định kỹ thuật
* **TanStack Query v5 typed generics**: `useQuery<DocumentsResponse>` explicit để `refetchInterval` callback truy cập `query.state.data.documents` type-safe. TanStack v5 `refetchInterval` nhận function `(query) => number | false`.
* **Query key strategy**: phân cấp `["tenants", tid, "workspaces", wid, "documents"]` — cho phép invalidate chọn lọc (`documentsKeys.list(tid,wid)`) ở T75 sau khi upload/delete. `tenantsKeys.all`, `workspacesKeys.list(tid)` export để reuse.
* **Tenant fallback qua Zustand**: mọi hook đọc `activeTenantId` từ `useTenantStore` khi caller không truyền `tid` — giữ call site (page T75) đơn giản, tự đồng bộ với header `X-Tenant-ID` của openapi-fetch middleware (T71). `enabled: Boolean(resolvedTid && wid)` để không fire khi thiếu context.
* **Polling status**: `PENDING_STATUSES = {"uploaded","processing"}` → poll 3s; khi toàn bộ `indexed|failed` → `refetchInterval: false` (dừng). Tránh request liên tục vô hạn.
* **Upload wrapper**: dùng `client.POST` với `body: FormData` — openapi-fetch 0.17 nhận FormData trực tiếp cho `multipart/form-data`, browser tự set boundary. Validate client-side 50 MiB trước khi gửi (FRONTEND_API_CONTRACT §4.2). `UploadValidationError` tách biệt với `ApiError` (network/server) để UI hiển thị message khác nhau.
* **Schema fallback cập nhật**: `schema.d.ts` là hand-maintained; bổ sung `status`/`filename`/`DocumentStatus`/preview/delete endpoint theo backend `DocumentItem` (backend `openapi/schemas.rs`). Khi backend online chạy `pnpm gen:api` để sync đầy đủ.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* `schema.d.ts` vẫn là fallback thủ công — T75 dùng field `status`/`filename`; nếu backend đổi tên field (snake vs camel) cần `pnpm gen:api` re-sync. Hiện `DocumentItem` backend trả `status: String` (serde lowercase enum) khớp `DocumentStatus` union.
* `useDocuments` poll 3s — nếu backend xử lý ingest chậm, polling kéo dài. UI T75 nên hiển thị badge "Processing" rõ ràng; cân nhắc max attempts sau này.
* `uploadDocument` không track upload progress (fetch API không có progress event). Nếu cần progress bar (contract §1333 gợi ý `XMLHttpRequest`), batch sau bổ sung wrapper riêng — hiện giữ `fetch`/openapi-fetch đơn giản.
* `ApiError` constructor signature `(status, code, message)` — upload.ts dùng `new ApiError(0, "upload-failed", "upload failed")` cho fallback no-data (status 0 vì không có HTTP status thật).
