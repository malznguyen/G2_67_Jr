#### Quyết định kỹ thuật
* **StatusBadge variant mapping**: dùng Badge cva variants `muted/warning/success/danger` thay vì hardcode màu — giữ nhất quán Zinc theme. Label qua `useTranslations("documents.status")` để Vi/En.
* **UploadDropzone**: client component, hỗ trợ cả drag-drop và click (accessibility: `role="button"` + keyboard Enter/Space). Validate client-side trước khi gọi API (giảm request lãng phí). `accept="application/pdf"` native filter. Dùng `UploadValidationError` vs `ApiError` để phân biệt lỗi validation vs server.
* **Documents page polling**: `useDocuments` (T74) tự `refetchInterval: 3000` khi có item `processing/uploaded`, `false` khi xong — UI tự cập nhật StatusBadge mà không cần manual refresh.
* **AclShareDialog mount**: chỉ render dialog khi `hasToken` (gate `session.access_token`, T70/T71). `aclConfig` memo từ `session.access_token` + `tid` + `NEXT_PUBLIC_API_BASE_URL`. `resourceType="document"`, `resourceId=doc.id`, `resourceLabel=doc.title`.
* **Delete flow**: `useMutation` gọi `client.DELETE("/tenants/{tid}/documents/{doc_id}")` + invalidate query list. Confirm dialog inline (không dep thêm) dùng dictionary `documents.deleteConfirm` + `common.cancel`. Error hiển thị `errors.deleteFailed`.
* **Async params (Next 16)**: page là client component nhưng nhận `params: Promise<...>`; unwrap bằng `React.use(params)` (Next 16 / React 19 hỗ trợ). Layout `[wid]` đã `await params` server-side.
* **Empty/loading/error states**: 3 nhánh rõ ràng dùng dictionary (`documents.empty`, `errors.loadFailed`). `ApiError` hiển thị kèm `error.code` để debug.

#### Blocker / Warning cho batch tiếp theo
⚠️ QUAN TRỌNG
* `uploadDocument` mặc định `visibility: "shared"` — hiện UI chưa cho user chọn visibility khi upload. Nếu cần toggle private/shared, batch sau thêm select trong UploadDropzone.
* Delete không có optimistic update — sau `invalidateQueries` list refetch đầy đủ. OK cho quy mô hiện tại; nếu list lớn cân nhắc `setQueryData` optimistic.
* AclShareDialog (T84) hiện hardcoded `bg-white dark:bg-neutral-900` (không dùng Zinc tokens) — vẫn hiển thị đúng nhưng lệch theme chuẩn. Batch sau (T77 design system) có thể refactor dialog dùng tokens.
* Polling 3s chỉ dừng khi toàn bộ `indexed|failed`. Nếu backend stuck `processing` (worker lỗi), polling kéo dài vô hạn — cân nhắc max attempts / timeout ở batch ops.
* Page dựa `session.access_token` cho ACL; nếu token hết hạn giữa session, nút Share sẽ 401 — cần refresh token flow (chưa có, T70 dùng access_token từ Keycloak account, không refresh).
