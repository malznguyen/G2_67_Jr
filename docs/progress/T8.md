#### Quyết định kỹ thuật

1. **App Router (không phải Pages Router)** — là chuẩn mặc định của Next.js 15; cũng là cơ sở cho route handler (`app/api/health/route.ts`) cần thiết cho `healthcheck` trong compose. Pages Router sẽ cần thêm `pages/api/health.ts` riêng và kém tương thích với RSC khi phát triển tiếp.
2. **Scaffold thủ công thay vì `create-next-app`** — `create-next-app` chạy tương tác, tải template lớn và hay "thêm" telemetry/ESLint config mặc định mà team chưa chuẩn hoá. Skeleton tối thiểu giúp T9+ có baseline rõ ràng để tinh chỉnh.
3. **`output: "standalone"`** — giảm image runtime xuống còn `server.js` + `.next/static` + `public`, tránh phải ship toàn bộ `node_modules`. Dockerfile chỉ copy 3 artifact này.
4. **Corepack + pnpm 10.32.1** — pnpm 10 sẵn cho Node 22; khoá version trong `package.json#packageManager` và `corepack prepare` để build reproducible.
5. **`context: ..` cho cả 3 build service trong compose** — Dockerfile dùng `COPY backend/...` / `COPY frontend/...`, tức là phải có repo root làm build context. Trước đó `context: .` (relative to compose file in `infra/`) tạo context sai thư mục, gây fail khi COPY. Kèm `.dockerignore` ở root để giữ context payload gọn (660B thay vì ~400MB).
6. **`app/api/health/route.ts`** — thêm route `/api/health` (dynamic, force-dynamic) để compose healthcheck `wget http://localhost:3000/api/health` pass khi `docker compose up`. Nếu không có route này, container sẽ liên tục fail healthcheck dù web server chạy được.
7. **Không cài Keycloak JS adapter / oidc-client ở task này** — để T70+ (frontend route) quyết định lib OIDC phù hợp; T8 chỉ cần skeleton boot được.
8. **React 19 RC (chứ không phải React 19.0.0 stable)** — Next 15.0.3 chính thức ghép với React 19 RC line `19.0.0-rc-66855b96-20241106`; nâng lên stable nên để ở batch frontend sau khi đã có use-case cụ thể.
9. **Không commit `pnpm-lock.yaml`?** — chọn **commit** lockfile để `docker compose build frontend` chạy `--frozen-lockfile` reproducible. Nếu thiếu lockfile thì fallback `pnpm install` ở Dockerfile sẽ resolve từ registry, dễ drift version giữa local dev và CI.

#### Blocker / Warning cho batch tiếp theo

⚠️ QUAN TRỌNG — batch sau phải đọc phần này trước khi bắt đầu

1. **`docker compose up` toàn stack**: tất cả 9 services giờ đã có Dockerfile tương ứng. Lệnh chuẩn từ repo root:
   ```bash
   cp .env.example .env
   docker compose -f infra/docker-compose.yml up -d
   ```
   Build context là repo root (`..` relative to `infra/docker-compose.yml`), nên đứng ở root và truyền `-f` rõ ràng. Build keycloak/ollama/qdrant/minio/redis/postgres từ image có sẵn (không cần Dockerfile), `backend`/`worker` dùng `infra/backend.Dockerfile`, `frontend` dùng `infra/frontend.Dockerfile`.

2. **Next 15.0.3 bị deprecation warning (CVE-2025-66478)**: hiện vẫn build và boot được. Nên lên lịch bump `next` lên patched version (≥ 15.4.x) trong batch frontend kế tiếp — chỉ thay đổi `frontend/package.json` rồi `pnpm install`.

3. **PowerShell execution policy**: trên máy dev hiện tại, `pnpm` / `npm` bị chặn do PS policy. Workaround đã dùng: `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass` rồi gọi `.cmd` trực tiếp (`& "C:\Users\admin\AppData\Roaming\npm\pnpm.cmd" ...`). Nếu team muốn `pnpm dev` thuần, cần fix policy một lần:
   ```powershell
   Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
   ```

4. **`corepack prepare pnpm@10.32.1`**: build Docker đã dùng `corepack` để activate đúng version. Nếu local dev chưa có `pnpm@10.32.1`, chạy `corepack enable && corepack prepare pnpm@10.32.1 --activate` một lần.

5. **Chưa thêm `oidc-client` / Keycloak JS adapter** — T9+ nếu chạm auth flow cần chọn lib và thêm vào `frontend/package.json`. Hiện tại các biến `NEXT_PUBLIC_KEYCLOAK_*` đã có sẵn trong `.env.example`/compose.

6. **Không có `NEXT_PUBLIC_TENANT_HEADER` env**: backend đang dùng `GMRAG_TENANT_HEADER` (line 58 `.env.example`). Khi frontend cần gắn tenant vào request, tốt nhất thêm `NEXT_PUBLIC_TENANT_HEADER` mirror để giữ contract khớp — đề xuất đưa vào batch T70+.

7. **`.env` ở root (local-only)**: đã tạo từ `.env.example` để compose không warn khi chạy `docker compose -f ... build`. `.env` đã có trong `.gitignore` (line 22), KHÔNG commit. Nếu clone fresh, dev phải tự `cp .env.example .env`.

8. **`.env.example` modification trước đó**: khi checkout thấy `.env.example` bị đổi `DEEPSEEK_MODEL=deepseek-chat` → `deepseek-v4-flash` (ngoài scope T8). T8 đã `git checkout -- .env.example` để commit T8 sạch. Batch sau kiểm tra xem sửa đó có thuộc task nào chưa, hoặc revert nếu sai.
