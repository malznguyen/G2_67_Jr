#### Quyết định kỹ thuật
- **Stack:** `utoipa 5` + `utoipa-swagger-ui 8` (không dùng v9 vì v9 kéo Axum 0.8, xung đột với Axum 0.7 của repo).
- **URLs:** Swagger UI `/swagger`, spec `/openapi.json` (public router, không cần JWT để xem docs).
- **Schemas:** Tập trung trong `openapi/schemas.rs` — một nguồn components OpenAPI; handlers giữ nguyên logic/DTO nội bộ.
- **Auth trong spec:** `SecurityScheme::Http` Bearer JWT; tenant-scoped paths document header `X-Tenant-Id` (phải khớp `{tid}`).
- **Tags:** 12 nhóm — Health, Users, Tenants, TenantMembers, Workspaces, WorkspaceMembers, Documents, ACL, Chat, Graph, Settings, Metering.

#### Audit findings (không đổi behavior)
| Finding | Severity | Ghi chú |
|---------|----------|---------|
| Không có success envelope thống nhất | Info | Mỗi endpoint trả collection key riêng (`tenants`, `documents`, …) |
| `GET /users/me` dùng `json!` | Low | Documented qua `MeResponse` schema |
| Health handlers dùng inline `json!` | Low | Documented qua `HealthResponse` / `HealthzResponse` |
| List endpoints không paginate | Info | Ghi trong mô tả API info |
| Domain values chủ yếu là `String` | Info | Enums OpenAPI mô tả allowed values |
| `429 quota-exceeded` | Info | Document trên upload |
| `503` healthz / JWKS | Info | Document trên healthz |
| Chat SSE hạn chế trên Swagger UI | Limitation | Dùng curl/EventSource để test stream đầy đủ |

#### Limitations
- Swagger UI "Try it out" cho SSE chat hiển thị stream hạn chế.
- Multipart upload cần chọn file thủ công trong UI.
- Docs luôn được serve (chưa có env flag tắt trên prod).
- OpenAPI path params `{tid}` vs Axum route `:tid` — utoipa chuẩn hóa sang `{tid}`.

#### Future improvements
- `OPENAPI_ENABLED=false` cho production.
- Refactor handlers dùng typed response structs (single Rust + OpenAPI source).
- Request/response examples trong spec.
- Generate TypeScript client từ `/openapi.json` cho frontend T70+.
- CI check: số operation OpenAPI == số route trong `lib.rs`.

#### Developer workflow
1. `cargo run -p gmrag-api` (cần Postgres, Redis, S3, Keycloak theo `.env`).
2. Mở `http://localhost:<port>/swagger`.
3. **Authorize** → paste Keycloak JWT.
4. Tenant routes: thêm header `X-Tenant-Id` (UUID tenant, khớp `{tid}` trong path).
5. Execute endpoints manually trước khi implement frontend.
